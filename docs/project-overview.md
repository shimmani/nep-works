# 프로젝트 개요

## 배경

엔이피(NEP)는 지자체(시/군/구)의 **시설물 유지관리** 사업을 수행하는 회사로, 입찰·수의계약을 통해 연간 20~50건의 프로젝트를 관리한다. 현재 모든 경리 업무를 **엑셀 수작업**으로 처리하고 있어, 반복적인 서류 작성과 정산 업무에 과도한 시간이 소요되고 있다.

## 목표

건설/시설관리업 경리 업무를 자동화하는 **Windows 데스크톱 앱**을 개발한다.

### 핵심 자동화 대상
1. **기성처리** — 기성내역서 작성, 증빙서류 취합, 진도율 자동계산
2. **준공서류** — 준공사진첩 자동 생성, 체크리스트 관리
3. **일용직 노무비** — 출역관리, 4대보험/원천징수 자동계산, 급여명세서
4. **입찰/계약 관리** — 나라장터 API 연동, 수의계약 한도 관리
5. **계획서 작성** — 사업계획서, 시공계획서 템플릿 기반 생성

## 사용자 환경

| 항목 | 내용 |
|------|------|
| 사용자 | 1명 (본인) |
| 작업 환경 | Windows PC |
| 기존 방식 | 엑셀 수작업 |
| 양식 | 내부 양식 + 지자체 지정 양식 혼용 |
| 연간 프로젝트 | 20~50건 |
| 일용직 규모 | 10~30명 |

## 기술 스택

| 영역 | 기술 | 선정 이유 |
|------|------|-----------|
| 프레임워크 | **Electron** | JS/TS 풀스택, 빠른 개발, 풍부한 엑셀 라이브러리 |
| 프론트엔드 | **React + TypeScript** | 생태계 최대, AI 코딩 지원 최적 |
| UI 라이브러리 | **Ant Design** | 테이블/폼/달력 등 경리 업무용 컴포넌트 풍부 |
| DB | **SQLite (better-sqlite3)** | 로컬 단일파일, 설치 불필요, 동기식 고성능 |
| 엑셀 엔진 | **ExcelJS** | 서식·병합셀·수식 지원, 템플릿 기반 생성 |
| 빌드 도구 | **electron-vite** | Electron 최적화 Vite 번들러 |
| 패키징 | **electron-builder** | Windows MSI/NSIS 인스톨러 생성 |

## 핵심 경쟁력

**지자체별 엑셀 양식 호환 + 자동 데이터 주입**

시장조사 결과, 준공서류 자동화 전용 솔루션은 시장에 거의 없으며, 기성처리 End-to-End 통합 솔루션도 부족하다. 기존 ERP(더존, 영림원)는 중소 시설관리업체에는 과도하고 비싸다. NEP-WORKS는 이 틈새를 1인 사용에 최적화된 경량 데스크톱 앱으로 채운다.
